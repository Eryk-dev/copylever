## Codebase Patterns
- Use `IF NOT EXISTS` for all CREATE TABLE, CREATE INDEX in SQL migrations
- Use `ADD COLUMN IF NOT EXISTS` for ALTER TABLE in migrations
- Supabase uses service_role key (bypass RLS) via `get_db()` singleton
- Auth dependencies: `require_user()` → `require_admin()` → `require_super_admin()`
- Session pattern: `secrets.token_urlsafe(32)`, 7 day expiry, stored in `user_sessions`
- Password hashing: `bcrypt.hashpw(password.encode("utf-8"), bcrypt.gensalt()).decode("utf-8")`
- Error messages in Portuguese (user-facing), English (logs/code)
- Python typecheck: `python3 -m py_compile <file>` or `python3 -c "import ast; ..."`
- Frontend typecheck: `cd frontend && npx tsc --noEmit`
