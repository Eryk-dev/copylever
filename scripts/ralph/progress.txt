- Use `python3` not `python` ‚Äî only python3 is available on this machine

- Shopee logistics channels are in `resp["response"]["logistics_channel_list"]`

- mypy errors in shopee_api.py are pre-existing (param type mismatches) ‚Äî ignore when checking new files
- Use `--ignore-missing-imports` with mypy for this project
- Shopee new item_id after add_item is at `resp["response"]["item_id"]`
- `api_debug_logs` table is shared between ML and Shopee ‚Äî use `action` field to differentiate
- `shopee_copy_logs.source_item_id` is bigint ‚Äî pass int, not string
- Supabase `.single()` raises on not-found ‚Äî no need for manual empty check
- `asyncio.gather(return_exceptions=True)` returns `BaseException` ‚Äî use `isinstance(r, BaseException)` for mypy
- Shopee item IDs are pure integers (not MLB-prefixed) ‚Äî normalize via `int()`
- Trial limit (check + increment) shares the same org counter for ML and Shopee
- `user_permissions` has no `platform` column ‚Äî slug uniqueness across ML/Shopee is sufficient, `platform` is computed in GET response
