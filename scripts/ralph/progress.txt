## Codebase Patterns
- In App.tsx, billing status is fetched via `GET /api/billing/status` with `auth.headers()` — reuse this pattern for any billing checks
- Paywall block is the conditional `if (billingAvailable && !paymentActive && !auth.user?.is_super_admin)` in App.tsx
- URL param cleanup uses `window.history.replaceState(null, '', window.location.pathname)`
- Design system vars: `--ink`, `--ink-muted`, `--surface`, `--space-*`, `--text-sm`, `--text-xs` etc.
- AuthView type controls which auth page is shown (login/signup)
- Migrations use `IF NOT EXISTS` for idempotency (tables and indexes)
- Migration files follow naming pattern: `NNN_description.sql` in `app/db/migrations/`
- Public auth endpoints (forgot/reset password) omit the `Depends()` param — no special decorator needed
- Auth pages (Login, Signup, ForgotPassword, ResetPassword) share identical card layout: centered flex, --surface bg, 380px max-width, animate-in class
- AuthView type in App.tsx controls which auth page renders: 'login' | 'signup' | 'forgot' | 'reset'
- Password reset tokens stored in `password_reset_tokens` table with 1h expiry, token via `secrets.token_urlsafe(32)`
